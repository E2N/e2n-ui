# Button

Ein Button führt eine Aktion über Tap oder Klick aus.

<Canvas of={ButtonStories.Basic} />

## Verwendung

### ✅ Use

- Wenn der Benutzer eine Aktion ausführen möchte z. B. etwas bestätigt.

### ❌ Don't use

- Wenn Du auf andere Seiten oder externe Ressourcen verlinken möchtest <br/> (verwende stattdessen eine Link-Komponente)
- Für eine hierarchische Navigation, bei der der Benutzer zu einem Unterthema <br /> navigiert und tiefer in das Menü eindringt (verwende stattdessen die Navigations-Elemente)

## Code

### Beispiel

```tsx
import React from 'react';

import Button from '@e2n/e2n-ui';

export function ButtonExample() => {
    return <Button variant="primary" onClick={() => handleClick()}>Primary Button</Button>
}
```

# Button vs. Icon Button

Die Komponente Button kann auch ein Icon enthalten jedoch immer nur vor oder hinter einem Text. Außerdem ist ein Button immer als Rechteck mit abgerundeten Ecken zu sehen, wobei hingegen das Icon Button immer einen Kreis umschließt.

# Button vs. Label

Die Komponente Button kann leicht mit einem Label verwechselt werden da sie ein ähnliches Aussehen besitzen. Ein Button besitzt im Gegensatz zum Label die 3 verschiedenen Zustände mit denen interagiert werden kann.

# Best Practice

✅ Verwende nach Möglichkeit Buttons nach ihren Typen (Variant, Color, Size, State)

✅ Benutze als Titel nur ein paar Wörter (idealerweise 1 oder 2)

✅ Starte mit einem Großgeschriebenen Verben, wie “Senden”

✅ Beschreibe die Aktion (Wenn jemand nur den Button liest, sollte er verstehen was danach folgt)

✅ Verwende die gleichen Worte wie der Titel des Kontexts

❌ Es gibt in der Regel in einer Ansicht nur einen Button als Variante Contained, da dieser die höchste Priorität besitzt und nur einmal vorkommt.

❌ Wähle für eine Bestätigung den Button nicht in den Farben Warning oder Error

❌ Weniger ist mehr, verwende nicht zu viele Buttons auf einem Screen
